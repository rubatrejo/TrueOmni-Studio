Bugs (6)

Rompen funcionalidad, regresiones, crashes. Click en cada card para detalles.

S-01 P0 ship-blocker Hydration mismatch en editor kiosk

Síntoma. Devtools console muestra "Hydration failed because the server rendered HTML didn't match the client. As a result this tree will be regenerated on the client" en cada carga del editor kiosk. El árbol del error apunta a <RedirectErrorBoundary> de Next.

Repro. Abrir cualquier /studio/<slug>/kiosk con devtools abierta. Reproducible 100%.

Impacto. React regenera el árbol entero client-side → flash de re-render visible → Lighthouse CLS/LCP cae → pueden surgir bugs sutiles en componentes con state client (ColorPicker, modales, autosave).

Fix propuesto. Auditar bootstrapStudioFromFs en SSR vs client; tipar timestamps que cambian entre passes (`Date.now()`, `new Date().toISOString()`). Probar a aislar a `` con `'use client'` puro y server component padre que solo pase props deterministas.

S-02 P1 alta postMessage con target origin '*'

Síntoma. Las 30+ funciones sendXNow de use-preview-bridge.ts usan win.postMessage(payload, '*').

Impacto seguridad. Si el iframe carga otra origin (rare en local pero posible en preview cross-domain o si el preview Vercel deploy tiene URL distinta), cualquier ventana puede leer branding/config sensibles. Best practice: target origin específico.

Fix propuesto. Constante IFRAME_ORIGIN = window.location.origin (mismo origin local) o configurable via env. Reemplazar todos los '*'.

Archivo. src/app/studio/_lib/use-preview-bridge.ts líneas 130-394 (71 ocurrencias de postMessage).

S-03 P1 alta client:list no se limpia al borrar cliente

Síntoma observado. En la sesión inicial, test-fase4 aparecía en dashboard con manifest pero sin cfg:test-fase4. La drift recovery (S-06) ya resuelve el caso GET, pero no purga client:list cuando un producto se borra parcialmente.

Impacto. Dashboard muestra clientes "fantasma". Si template default deja de existir, los fantasmas dan 404 permanente.

Fix propuesto. En DELETE /api/studio/configs/[slug], verificar manifest y purgar client:list + client:{slug}:branding + client:{slug}:manifest cuando ya no quedan productos activos.

S-04 P2 media Console errors silenciosos en bridge

30+ try { postMessage } catch (e) { console.warn('[bridge:postMessage]', e) } idénticos. Si el iframe está dead-locked, el operador no ve feedback visible. Solo en devtools.

Fix. Helper postToIframe(type, payload); si 5+ fallos consecutivos, banner "Live preview disconnected — reload" arriba del iframe.

S-05 P2 media Photo Booth tab dispara console errors

Repro. Editor kiosk → click Photo Booth tab. Devtools muestra +3 console errors.

Fix. Investigar PhotoBoothEditor; posible cleanup de stream MediaPipe missing al unmount.

S-06 P3 nice-to-have Drift recovery no rellena client:list

Mi commit d3fd17f hace kv.set(cfg) pero no kv.sadd(clientsList). Auto-migrate corrige en el siguiente GET → convergente, pero hay ventana donde el dashboard no lo lista.

Fix. Añadir await kv.sadd(kvKeys.clientsList, slug) tras la materialización en /studio/[slug]/kiosk/page.tsx y /studio/[slug]/digital-displays/page.tsx.

Gaps funcionales (8)

S-07P1No hay search en dashboard
Con 4 clientes ya escala mal. Añadir <input type="search"> arriba del grid + filter local. Pattern: similar al search del modal Languages.
S-08P1No hay rollback de unified branding
Branding autosave 1s después del cambio. Si el operador rompe el primary sin querer no hay "Discard". Solo undo del editor kiosk individual. Fix: botón "Restore" que recargue de KV o snapshots paralelos a kiosk.
S-09P1Activate falla si default no existe
Activate clona desde default. Si lo borran por error, todo activate falla. Fix: marcar default reservado/inmutable; o fallback a empty config + warning.
S-10P2Modal NewClient sin DD checkbox
Modal solo crea kiosk. Para tener cliente con DD desde el inicio hay que crear y luego activar. Fix: checkboxes "Activate kiosks (default)" + "Activate digital displays" antes de submit.
S-11P2No hay diff pre-publish unificado
Publish está en kiosk + signage por separado. La Vista de Cliente no consolida un "Publish all products" con diff de qué cambia. Fix: sección "Pending changes" en /studio/[slug].
S-12P3Duplicate solo clona kiosk
Hover muestra Duplicate; endpoint /clone solo clona el kiosk. Manifest + signage + unified branding no se replican. Fix: POST /api/studio/clients/[slug]/clone que clone toda la entidad.
S-13P3Sin pinning / recently edited
Dashboard ordena alfabético + default primero. Sin pin/star para clientes top. Fix: flag pinned en manifest + sort por pinned > recent > alpha.
S-14P3Stubs Coming Soon sin contenido
/mobile-pwa, /video-walls, /tablets renderizan placeholder. Sin teaser, lista waitlist, link a roadmap. Fix: card con descripción + link a doc + opción "Notify me when ready".

Friction UX (9)

S-15P1Tres patrones de tabs distintos

BrandingForm usa <button aria-current> sin tablist. Editor kiosk usa role=tablist+role=tab. Editor signage usa el mismo patrón que kiosk. Visual también difiere (chip vs underline vs glow).

Fix. Componente <TabStrip> reutilizable con role=tablist + variants visuales.

S-16P2Footer "Local · main" hardcoded
En producción Vercel mostraría lo mismo. Confuso. Fix: derivar de VERCEL_ENV + VERCEL_GIT_COMMIT_REF.
S-17P2Color picker sin click-outside
BrandingForm Brand colors → click swatch → picker abre. Solo cierra con "Done" o re-toggle. Fix: useClickOutside ref que dispatcha setOpen(false).
S-18P2No valida rangos lat/lon
Latitude acepta -200, Longitude 999. Solo number-check. Fix: clamp a [-90,90] / [-180,180] con error en helper text.
S-19P2"Coming soon" sin diferenciación
PWA / Walls / Tablets / "Add display" usan badge idéntico pero con timelines distintas. Fix: variants — "Coming soon · Q3 2026", "In development", "Next sprint".
S-20P2Activate visualmente igual a Open
Ambos usan flecha →. Activate clona desde default (operación pesada) — debería sentirse distinto. Fix: estilo "ghost" + icon Plus para Activate, ArrowRight para Open editor.
S-21P3Sticky header overlapa first field
En viewport <900px el sticky banner cubre el primer Field. Fix: padding-top dinámico en el panel.
S-22P3Status pill "Idle" confuso
Vista de Cliente muestra "Idle" cuando no hay cambios — confuso ("editor is idle" vs "I should do something"). Fix: copy "Saved" si hay branding persisted, "—" si vacío.
S-23P3Cancel sin confirm en BrandingForm
ClientView tiene beforeunload para cierre completo. Pero navegación interna (link a /studio o /kiosk) no dispara confirmación. Fix: Next router beforePopState o link interceptor.

Accesibilidad (8)

S-24P22 botones "+ New client" idénticos
Header pill + card dashed al final del grid. Strict mode de Playwright lo detecta. Fix: aria-label distinto en cada uno.
S-25P2Accessible names verbosos en cards
"TrueOmni default Arizona Kiosks Displays Edited 1h ago ruben" — concatena logo alt + slug + heading + badges + timestamp. Fix: subcontenido aria-hidden="true"; aria-label="Open client {name}, {N} products active" al Link.
S-26P2Status badge con métricas en aria
"All systems operational" link tiene aria-label="KV: ok (cloud, 591ms) Filesystem: ok (4ms)..." — métricas embebidas. Fix: texto visible + aria-hidden en detalles.
S-27P2BrandingForm tabs sin role tablist
Screen reader anuncia "button" en lugar de "tab N of 5". Fix: convertir a role="tablist" + role="tab" consistente con editor kiosk/signage.
S-28P2Escape inconsistente en modales
Audit anterior F-35 lo abrió. Hoy NewClientModal sí cierra con Escape; falta verificar DeleteKiosk, DuplicateKiosk, AiSuggest, Publish, AddLanguage. Fix: extraer useEscapeClose hook + aplicar a todos.
S-29P3Sin focus trap en modales
Tab/Shift+Tab dentro del modal puede salir al body. Fix: <FocusTrap> wrapper para todos los modales del Studio.
S-30P3aria-live faltante en SaveStatusPill
"Saving…/Saved/Idle/Error" cambian sin notificar a screen readers. Fix: wrapper aria-live="polite" o role="status".
S-31P3Inputs sin htmlFor explícito
BrandingForm <Field> envuelve <input> en <label>. Válido por wrapping pero menos robusto. Fix: useId() + htmlFor explícito.

Performance (6)

S-32P125 useEffects en Shell.tsx
Cada sección del config tiene useEffect(() => pushX(X), [X, pushX]). Cualquier cambio dispara su effect → 25 postMessages individuales al iframe. Fix: single useEffect con diff o store-driven con selectors.
S-33P1Shell.tsx 1072 líneas — render N×
Componente monolítico con state global. React re-renders todo Shell en cada cambio. Fix: split en <EditorPanels> (per-tab) memoized + <PreviewPanel> con su propio store.
S-34P2Auto-migrate loop secuencial
GET /api/studio/clients hace Promise.all para summaries. Pero antes corre auto-migrate con loop secuencial (line 76 de auto-migrate-clients.ts). Fix: Promise.allSettled con concurrency limit (e.g. 5).
S-35P2Iframe no se desmonta al salir
Navegar de /studio/default/kiosk a /studio mantiene memoria del iframe. Fix: useEffect cleanup que pone iframe.src = 'about:blank'.
S-36P3Logos sin loading="lazy"
Dashboard muestra logos como <img> sin loading="lazy" ni dimensiones. Fix: loading="lazy" + decoding="async" + width/height para CLS.
S-37P3Auto-migrate corre cada GET
Lazy on-load, idempotente, pero costoso (escanea FS + KV cada vez). Fix: cache TTL 60s en kv.get('migration:lastRun').

Automatización / DX (6)

S-38P1Sync hook fail silencioso
Mi commit añadió try/catch que loguea pero no aborta el create. Si falla, cliente queda sin manifest → auto-migrate lo recupera al siguiente GET. Intencional pero invisible. Fix: logger estructurado + alerta si falla > 1 vez.
S-39P2Sin tests E2E del flujo
Validación manual con Playwright esta sesión. No hay test commiteado. Fix: tests/e2e/studio-create-client.spec.ts con Playwright + setup de KV mock.
S-40P2format:check con drift histórico
331 archivos con drift de prettier acumulado. Fix: commit one-shot pnpm format + bloquear via pre-commit hook.
S-41P3Docs/changelog hardcoded
Mismo hallazgo del audit anterior — entries hardcoded; no genera desde GitHub Releases. Diferido a S7.2.
S-42P3client:list sin observability
Sin logs estructurados de mutaciones (add/remove). Difícil debug del drift de S-03. Fix: wrapper addClientToList con logger.info({event:'client.added',slug,by}).
S-43P3Auto-migrate sin report visible
MigrationReport se devuelve pero solo se loguea en server. Operador no ve "2 clientes migrados". Fix: banner toast + sección en /studio/diagnostics.
