## Purpose

This is the single most comprehensive UI/UX reference document for the TrueOmni website project. It provides instant access to 50 design styles, 21 color palettes, 50 font pairings, layout patterns, spacing systems, accessibility guidelines, and micro-interaction patterns. Use it to make informed, consistent, and high-quality design decisions.

## 5. Design Guidelines

### Spacing System (4px Base, 8pt Grid)

```
Base unit: 4px
Scale:
  0:   0px
  0.5: 2px
  1:   4px
  1.5: 6px
  2:   8px    ← minimum touch target padding
  3:   12px
  4:   16px   ← base paragraph spacing
  5:   20px
  6:   24px   ← standard component padding
  8:   32px   ← section inner padding
  10:  40px
  12:  48px   ← between sections (mobile)
  16:  64px   ← between sections (desktop)
  20:  80px
  24:  96px   ← large section spacing
  32:  128px  ← hero spacing
  40:  160px
  48:  192px

8pt Grid Rules:
- All spacing values should be multiples of 8px for consistency
- Exception: 4px for tight spacing (icon padding, borders)
- Container max-width: 1280px (80rem)
- Container padding: 16px mobile, 24px tablet, 32px desktop
- Column gap in grids: 16px mobile, 24px tablet, 32px desktop

Modular Scale for Type Spacing:
- Line height body: 1.5 - 1.6
- Line height headings: 1.1 - 1.3
- Paragraph margin-bottom: 1em
- Heading margin-bottom: 0.5em
- Section padding-y: 64px mobile, 96px tablet, 128px desktop
```

### Color Theory — The 60-30-10 Rule

```
60% — DOMINANT COLOR (background, large surfaces)
  → Usually white/light or dark/neutral
  → Sets the overall mood of the site
  → Example: White (#FFFFFF) or Dark (#0F172A)

30% — SECONDARY COLOR (containers, cards, sections)
  → Supporting color that creates visual interest
  → Used for alternate section backgrounds, cards, navigation
  → Example: Light gray (#F8FAFC) or darker shade (#1E293B)

10% — ACCENT COLOR (CTAs, links, highlights)
  → Your brand primary color
  → Used sparingly for maximum impact
  → Buttons, links, active states, badges, icons
  → Example: Brand blue (#1E40AF) or brand purple (#7C3AED)

Application to TrueOmni:
  Page background:  60% → white/light neutral
  Cards/sections:   30% → light gray or brand tint
  CTAs/highlights:  10% → primary brand color
```

### WCAG Contrast Guidelines

```
WCAG AA (minimum requirement — our target):
  Normal text (< 18px / < 14px bold):     4.5:1 contrast ratio minimum
  Large text (>= 18px / >= 14px bold):     3.0:1 contrast ratio minimum
  UI components and graphical objects:      3.0:1 contrast ratio minimum

WCAG AAA (enhanced — aspirational):
  Normal text:                              7.0:1 contrast ratio minimum
  Large text:                               4.5:1 contrast ratio minimum

Contrast Ratio Formula:
  L1 = lighter color relative luminance
  L2 = darker color relative luminance
  Ratio = (L1 + 0.05) / (L2 + 0.05)

  Relative Luminance:
  L = 0.2126 * R + 0.7152 * G + 0.0722 * B
  where R, G, B are linearized:
    if sRGB <= 0.03928: linear = sRGB / 12.92
    else: linear = ((sRGB + 0.055) / 1.055) ^ 2.4

Quick Reference — Common Pairs on White (#FFFFFF):
  #000000 (black):        21.0:1  ← AAA pass
  #1E293B (slate-800):    12.5:1  ← AAA pass
  #374151 (gray-700):      9.5:1  ← AAA pass
  #6B7280 (gray-500):      4.6:1  ← AA pass (normal text)
  #9CA3AF (gray-400):      2.9:1  ← AA pass (large text only)
  #D1D5DB (gray-300):      1.7:1  ← FAIL (decorative only)

Rules:
  ✅ Body text: always use AAA ratio (7:1+) for maximum readability
  ✅ Headings: AA ratio (4.5:1) minimum, prefer AAA
  ✅ Captions/labels: AA minimum (4.5:1 for normal, 3:1 for large)
  ✅ Buttons: button text must meet AA on button background
  ✅ Links: must be distinguishable from surrounding text (not just color)
  ✅ Focus indicators: 3:1 contrast against adjacent colors
  ❌ Never rely on color alone to convey meaning
  ❌ Never use light gray text on white for important content
```

### Typography Scale

```
Modular Scale Ratios:

  Major Second:     1.125   (subtle, for data-heavy layouts)
  Minor Third:      1.200   (balanced, for content sites) ← RECOMMENDED
  Major Third:      1.250   (clear, for marketing sites) ← RECOMMENDED
  Perfect Fourth:   1.333   (dramatic, for bold statements)
  Augmented Fourth: 1.414   (very dramatic, for editorial)
  Perfect Fifth:    1.500   (extreme, for hero-focused sites)

Example Scale with Major Third (1.250) and Base 16px:
  xs:     10.24px  (0.64rem)   — fine print, labels
  sm:     12.80px  (0.80rem)   — captions, meta text
  base:   16.00px  (1.00rem)   — body text
  lg:     20.00px  (1.25rem)   — lead paragraphs, subheadings
  xl:     25.00px  (1.563rem)  — H4
  2xl:    31.25px  (1.953rem)  — H3
  3xl:    39.06px  (2.441rem)  — H2

### Responsive Breakpoints Strategy

```
Breakpoints (Tailwind v4 defaults):
  sm:   640px    — Large phones landscape
  md:   768px    — Tablets portrait
  lg:   1024px   — Tablets landscape / small laptops
  xl:   1280px   — Desktops
  2xl:  1536px   — Large desktops

Mobile-First Approach:
  1. Design for 375px width first (iPhone SE)
  2. Add sm: adjustments for 640px+
  3. Add md: adjustments for 768px+ (significant layout changes)
  4. Add lg: adjustments for 1024px+ (multi-column layouts)
  5. Add xl: adjustments for 1280px+ (max-width container kicks in)

Container Strategy:
  Mobile:   full-width with 16px padding
  Tablet:   full-width with 24px padding
  Desktop:  max-width 1280px centered with 32px padding

Grid Columns:
  Mobile:   1 column (stack everything)
  Tablet:   2 columns (some side-by-side)
  Desktop:  3-4 columns (full grid layouts)
  Large:    4-6 columns (data-dense layouts only)

Typography Responsive Scaling:
  Mobile:   base 16px, headings scale down 1 step
  Tablet:   base 16px, headings at designed scale
  Desktop:  base 16-18px, headings at full scale
  Large:    base 18px, headings can go larger
```

### Micro-Interaction Patterns

```
HOVER EFFECTS:
  Button hover:     background-color shift (150ms ease), slight scale(1.02)
  Card hover:       shadow increase (200ms ease), translateY(-2px)
  Link hover:       underline slide-in (200ms ease), color shift
  Image hover:      scale(1.05) with overflow hidden (300ms ease)
  Icon hover:       color shift + slight rotate(5deg) (200ms ease)
  Nav item hover:   background-color pill (150ms ease)

CLICK/TAP EFFECTS:
  Button click:     scale(0.98) for 100ms, then spring back
  Card click:       subtle scale(0.99) (100ms ease)
  Toggle click:     spring animation for switch (300ms spring)
  Like/Favorite:    scale(1.2) bounce then settle (400ms spring)

SCROLL EFFECTS:
  Fade up:          opacity 0→1 + translateY(20px→0) on scroll into view
  Stagger fade:     children animate sequentially with 50-100ms delay each
  Parallax:         background moves at 0.5x scroll speed
  Progress bar:     width tied to scroll position
  Sticky header:    shrink + shadow on scroll down, expand on scroll up
  Counter up:       numbers count up when stats section enters viewport

LOADING EFFECTS:
  Skeleton:         shimmer animation on placeholder rectangles (1.5s infinite)
  Spinner:          rotate animation (0.75s linear infinite)
  Progress:         determinate bar with % (ease-out transitions)
  Content fade:     fade-in from skeleton to real content (300ms ease)
  Image blur:       blur(20px)→blur(0) as image loads (500ms ease)

TRANSITION STANDARDS:
  Duration:
    Micro (hover, focus):     100-200ms
    Small (expand, collapse): 200-300ms
    Medium (page, modal):     300-500ms
    Large (complex sequences): 500-1000ms

  Easing:
    ease-out:    most transitions (entering elements)
    ease-in:     exiting elements
    ease-in-out: elements changing state
    spring:      playful/bouncy interactions (Framer Motion)
    linear:      continuous animations (spinners, progress)

  Reduce Motion:
    ALWAYS respect prefers-reduced-motion:
    @media (prefers-reduced-motion: reduce) {
      *, *::before, *::after {
        animation-duration: 0.01ms !important;
        transition-duration: 0.01ms !important;
      }
    }
```

### Visual Hierarchy Principles

```
HIERARCHY TOOLS (strongest to weakest):
  1. SIZE:        Larger elements are seen first
  2. COLOR:       High contrast / brand color draws attention
  3. WEIGHT:      Bold text stands out from regular text
  4. POSITION:    Top-left (LTR) is scanned first (F-pattern, Z-pattern)
  5. WHITESPACE:  Isolated elements command more attention
  6. DEPTH:       Elevated elements (shadows) feel closer/more important

F-PATTERN (content pages):
  Users scan horizontally across the top
  Then scan horizontally across the middle
  Then scan vertically down the left side
  → Place key content and CTAs along these lines

Z-PATTERN (landing pages):
  Eye moves: top-left → top-right → bottom-left → bottom-right
  → Logo top-left, CTA top-right or bottom-right

VISUAL WEIGHT RANKING (heaviest to lightest):
  1. Hero headline (4xl-6xl, bold, dark)
  2. CTA buttons (color, size, isolation)
  3. Section headings (2xl-3xl, semibold)
  4. Images and illustrations
  5. Subheadings (lg-xl, medium weight)
  6. Body text (base, regular weight)
  7. Captions and metadata (sm, light color)

WHITESPACE RULES:
  More space around an element = more importance
  Related elements have less space between them (proximity principle)
  Unrelated elements have more space between them
  Section spacing > component spacing > element spacing
  Never fear "too much" whitespace — it's a luxury signal
```

```

### Shadow Scale

```
shadow-xs:    0 1px 2px 0 rgb(0 0 0 / 0.05)

  Cards at rest:        shadow-sm

### Border Radius Scale

```
rounded-none:   0px      — Tables, data grids

## 7. Design Audit Checklist

Use this to review any page or component before shipping:

```
LAYOUT:
□ Consistent spacing (8pt grid)
□ Proper alignment (nothing looks "off")
□ Visual hierarchy is clear (you know where to look first)
□ Responsive on mobile (375px), tablet (768px), desktop (1280px)
□ Content doesn't overflow or clip unexpectedly

COLOR:
□ Follows 60-30-10 rule
□ All text passes WCAG AA contrast (4.5:1 normal, 3:1 large)
□ Interactive elements are clearly distinguishable
□ Color is not the only indicator of meaning
□ Consistent with design system palette

TYPOGRAPHY:
□ Consistent font family usage (heading vs body)
□ Correct font weights per hierarchy level
□ Line height appropriate (1.5+ for body, 1.1-1.3 for headings)
□ No orphans in headlines (last line has at least 2 words)
□ Max line width: 65-75 characters for body text

COMPONENTS:
□ All states designed (default, hover, focus, active, disabled)
□ Focus indicators visible (keyboard navigation)
□ Touch targets minimum 44x44px on mobile
□ Consistent border radius throughout
□ Consistent shadow usage

ACCESSIBILITY:
□ Alt text on all images
□ ARIA labels on interactive elements
□ Keyboard navigable (Tab, Enter, Escape)
□ Focus order logical (left-to-right, top-to-bottom)
□ prefers-reduced-motion respected
□ prefers-color-scheme respected (if dark mode)

PERFORMANCE:
□ Images optimized (WebP, proper sizing)
□ No layout shift (CLS < 0.1)
□ Above-fold content loads fast (LCP < 2.5s)
□ Fonts loaded efficiently (display: swap)
□ Animations use transform/opacity only (GPU-accelerated)
```
